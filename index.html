HHKB使用感受
  • hhkb
  • linux

    前几天国庆节的时候入手了HHKB，到手的时候说实话并没有什么特别的感觉，键帽的触感不错。按起来跟之前的茶轴感觉不一样，一开始的阻力比较大，声音有点发闷，整体感觉很小，很轻。携带起来很方便。之前的酷冷xt虽然是87的，但是里面有钢板，很沉。这次就算是一步到位了，希望自己能退烧。
    通过调节背后的开关，可以把左边的Meta键变成Fn，把右上角的Delete换成使用比较频繁的Backspace。用了一会感觉还是别扭，主要是方向键。于是在网上搜了一下，发现在OSX下的一个神器–Karabiner。通过它我把Control + jkli映射成了左下右上。不过公司的电脑用的是Arch，怎么办？找了一下发现通过xbindkeys+xvkbd能够实现类似的效果。
    安装好xbindkeys和xvkbd之后，打开

~/.xbindkeysrc

添加如下配置：

"/usr/bin/xvkbd -xsendevent -text \\[Left]"
  Control + j

"/usr/bin/xvkbd -xsendevent -text \\[Right]"
  Control + l

"/usr/bin/xvkbd -xsendevent -text \\[Up]"
  Control + i

"/usr/bin/xvkbd -xsendevent -text \\[Down]"
  Control + k

运行xbindkeys ，就给可以用

Xcode添加空工程模板

发表于 2015-10-07

Xcode6之后默认无法创建Empty Application.
下载模板
https://github.com/jymsy/AddMissingTemplates

将Empty Application.xctemplate文件夹存放到
/Applications/Xcode.app/Contents/Developer/Platforms/iPhoneOS.platform/Developer/Library/Xcode/Templates/Project Templates/iOS/Application

重启Xcode之后就可以使用了。

sublime text打开文件时总是新建窗口

发表于 2015-10-04

发现在mac下sublime text总是在新窗口中打开文件，很麻烦。
后来才发现通过设置”open_files_in_new_window”: false，就可以了。

具体设置：Preferences -> Settings – User -> 添加open_files_in_new_window，将其设为false。然后重启sublime text。
